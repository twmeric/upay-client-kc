## 🔄 狀態變更流程

| 步驟 | 觸發條件 | 狀態變更 | 誰明 |
|------|---------|---------|------|
| 1 | 用戶點擊支付 | `pending` | 訂單創建，等待支付 |
| 2 | EasyLink Webhook | `paid` | 支付成功，更新狀態 |
| 3 | EasyLink Webhook | `failed` | 支付失敗 |
| 4 | 超時（30分鐘）| `expired` | 訂單過期（需要定時任務）|

## ⚠️ 當前問題

### 問題 1: Webhook 可能未正確配置
**影響**：支付完成後狀態不會自動更新

**解決方案**：
1. 聯繫 EasyLink 客戶經理配置 Webhook
2. 或使用輪詢 API 手動更新狀態

### 問題 2: 定時狀態同步缺失
**影響**：已支付訂單可能顯示為「處理中」

**解決方案**：
1. 添加輪詢任務，定期檢查訂單狀態
2. 或實現前輪輪詢機制

## ✅ 驗證交易追蹤

檢查交易是否正常追蹤：

1. **創建訂單** - 檢查是否保存到 DB（status: pending）
2. **支付完成** - 檢查 Webhook 是否收到
3. **狀態更新** - 檢查是否變為 paid
4. **前端顯示** - 刷新頁面查看狀態變更
